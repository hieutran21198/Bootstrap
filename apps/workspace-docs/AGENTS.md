# apps/workspace-docs/

## OVERVIEW

Docusaurus 3.10 static site (React 19, TypeScript) that renders workspace docs as a public, read-only GitHub Pages project site: `https://hieutran21198.github.io/Bootstrap/`.

This app authors no canonical doc content. Two `@docusaurus/plugin-content-docs` instances read sibling trees in place: workspace docs from `../../docs` at route `/docs` with landing `/docs/AGENTS`, and portal docs from `../../services/portal/docs` at route `/portal`.

## STRUCTURE

```
apps/workspace-docs/
├── docusaurus.config.ts     # two docs instances, navbar/footer, mermaid, prism, docRoots
├── sidebarsWorkspace.ts     # autogenerated workspace docs sidebar
├── sidebarsPortal.ts        # autogenerated portal docs sidebar
├── plugins/
│   └── remark-repo-links.mjs # rewrites repo-relative markdown links at build time
├── src/
│   ├── pages/index.tsx      # homepage
│   └── css/custom.css       # Infima theme customizations
├── static/                  # static assets
├── devenv.nix               # JS/npm + core.ai slim + markdown toolchain
└── package.json             # npm scripts: start/build/serve/typecheck
```

## WHERE TO LOOK

| Need | Location |
|------|----------|
| Add/edit workspace documentation | `../../docs/` |
| Add/edit portal documentation | `../../services/portal/docs/` |
| Change rendered doc roots/routes | `docusaurus.config.ts` (`plugin-content-docs` instances + `docRoots`) |
| Change sidebar behavior | `sidebarsWorkspace.ts`, `sidebarsPortal.ts` |
| Change repo-relative link rewriting | `plugins/remark-repo-links.mjs` |
| Change homepage/theme | `src/pages/index.tsx`, `src/css/custom.css` |
| Add static assets | `static/` |
| Run docs locally | `docs-install`, `docs-dev`, `docs-build`, `docs-serve` from this devenv |

## CONVENTIONS

- Docs content is not written here. Edit the real files under root `docs/` and `services/portal/docs/`; this app only renders them.
- Links in source docs stay repo-relative so they work on GitHub and in-repo. `plugins/remark-repo-links.mjs` rewrites them at build time to in-tree site routes or to `github.com` blob/tree URLs for repo files outside rendered doc trees; source docs are never modified.
- New top-level doc trees require a new `@docusaurus/plugin-content-docs` instance, sidebar wiring, and a `docRoots` entry in `docusaurus.config.ts`.
- Use npm scripts directly (`start`, `build`, `serve`, `typecheck`) or the scoped devenv wrappers (`docs-dev`, `docs-build`, `docs-serve`).

## ANTI-PATTERNS

- Do not hand-edit generated `.docusaurus/` or `build/` directories; they are gitignored build outputs.
- Do not duplicate or author canonical docs inside this app.
- Do not hardcode absolute site paths in source docs; keep repo-relative links and let the remark plugin resolve them.
- Do not add a doc tree without wiring its plugin instance, sidebar, and `docRoots` entry.
- Do not touch generated `CLAUDE.md` (`@AGENTS.md`) in this directory.
