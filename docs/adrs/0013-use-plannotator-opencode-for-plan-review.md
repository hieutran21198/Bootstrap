# 0013. Use Plannotator OpenCode plugin for orchestrated plan review

- **Status**: Accepted
- **Date**: 2026-07-03
- **Deciders**: Minh Hieu Tran <hieu.tran21198@gmail.com>
- **Supersedes**: -
- **Superseded by**: -

## Context

The workspace's AI system is generated by Nix/devenv: `packages/nix/core/ai/opencode/default.nix` owns the OpenCode `plugin` list, while `packages/nix/core/ai/default.nix` renders per-agent files and a default-deny permission map for managed MCPs and skills. ADR-0007 already commits the workspace to Nix-owned generated config, devenv runners, and no ad-hoc `setup.sh` / `curl | bash` install path.

We want a human-visible review gate for agent plans before implementation. The third-party Plannotator OpenCode plugin (`backnotprop/plannotator`, npm package `@plannotator/opencode`) provides a `submit_plan` tool and workflow modes that can constrain the tool to planning agents. Its documented OpenCode install uses the npm plugin reference `@plannotator/opencode@latest`; slash commands and CLI fallback depend on a separately installed `plannotator` binary from the upstream installer.

The decision must stay thin: record the OpenCode plugin configuration and policy, but leave the Nix edit to an implementer.

## Decision

We will wire Plannotator as an OpenCode plugin for **plan review only**, using the upstream npm plugin reference, `plan-agent` workflow, and the workspace's orchestrator as the workspace planning agent.

The implementer will apply this `plugin` array in `packages/nix/core/ai/opencode/default.nix`:

```nix
plugin = [
  "compound-engineering@git+https://github.com/EveryInc/compound-engineering-plugin.git"
  [
    "@plannotator/opencode@latest"
    {
      workflow = "plan-agent";
      planningAgents = [
        "plan"
        "orchestrator"
      ];
    }
  ]
];
```

Runtime is intentionally omitted: Plannotator auto-detects runtime (embedded under Bun-hosted OpenCode), and only `cli` is a documented explicit override.

Details:

1. **Plugin reference**: use npm `"@plannotator/opencode@latest"`, not a git ref. This is the plugin's documented OpenCode distribution and avoids coupling this workspace to the upstream repository's build layout.
2. **Workflow**: use `"plan-agent"`. It exposes `submit_plan` at the planning boundary without nudging broad build/implementation agents.
3. **Planning agents**: list OpenCode's built-in `"plan"` agent explicitly to preserve the upstream default, and add the workspace `"orchestrator"` because it is the only `mode = "primary"` planning/routing agent. No subagent belongs in this list.
4. **Binary and slash commands**: defer the `plannotator` binary and `/plannotator-*` commands. We will not run the upstream `curl | bash` installer. If a runtime path requires the CLI or slash commands later, the follow-up is to package the binary declaratively through Nix with verification, not to install it out of band.
5. **Governance**: record this as an ADR. Adding a third-party OpenCode plugin changes the AI tool surface and supply-chain posture, so a config change plus an inline comment or finding is not enough. Findings remain for investigations; this is a durable decision.
6. **Permissions**: keep the Nix renderer as-is. Its default-deny map only covers managed MCPs and skills and has no `*` catch-all for plugin tools, so it should not pre-deny Plannotator's `submit_plan`. Plannotator's `plan-agent` mode is responsible for allowing `submit_plan` for `plan`/`orchestrator` and denying it for non-planning primary agents.

## Consequences

- **Positive**:
  - Plan review becomes available at the orchestrated planning boundary without broadening every agent to a plan-submission tool.
  - The plugin uses its documented OpenCode package form, so the configuration matches upstream support and examples.
  - The binary installer remains outside the workspace until it can be represented as Nix-governed, reviewable supply-chain state.
- **Negative**:
  - `@latest` is a moving upstream target. If plugin drift causes breakage, a follow-up should pin an exact npm version or package the plugin through Nix.
  - Runtime selection is upstream-owned auto-detection; non-Bun or wrapped OpenCode hosts may require a future Nix-packaged CLI path.
  - Slash-command workflows (`/plannotator-review`, `/plannotator-annotate`, `/plannotator-last`) are unavailable until a Nix-packaged binary/command story exists.
- **Neutral**:
  - Plannotator may add effective OpenCode permission entries for `submit_plan` and markdown editing for planning agents. Runtime verification must confirm the final merged permissions are still least-privilege for this workspace.

## Alternatives considered

- **Reference the plugin by git URL, matching the existing compound-engineering entry.** Rejected because Plannotator's documented OpenCode install is the npm package. A git ref would make this workspace depend on upstream repository structure and build outputs that the plugin author does not present as the primary install surface.
- **Use `manual` workflow.** Rejected because it does not register `submit_plan`, which is the plan-review integration we want now.
- **Use `user-managed` workflow.** Rejected for this thin wiring step because it would require new Nix renderer support or hand-authored per-agent permission entries for a plugin-owned tool. Without that extra work, it risks either no planning prompt or an under-specified permission boundary.
- **Use `all-agents` workflow.** Rejected because it is the broad legacy behavior and conflicts with the workspace's orchestrator-centric, default-deny posture.
- **Install the `plannotator` binary and slash commands with `curl | bash`.** Rejected because ADR-0007 makes devenv scripts and Nix-owned generated config the only runner/install path. If slash commands become required, package the binary declaratively in Nix and set any needed environment such as `PLANNOTATOR_BIN` from the Nix module.
- **Treat this as only a config change plus a finding.** Rejected because the decision introduces a third-party agent plugin and changes the AI system's tool surface; that is an architecture decision, not just investigation evidence.
