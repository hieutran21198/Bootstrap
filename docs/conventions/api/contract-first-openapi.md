# Contract-first OpenAPI

> **Scope**: workspace-wide — every REST API contract under `services/*/api/*.yaml`, the generated Go HTTP delivery boundary under `services/*/internal/delivery/http/...`, and the hand-written delivery adapters that mount generated routes for those contracts.
> **Status**: Active
> **Decided by**: [ADR-0018](../../adrs/0018-contract-first-rest-api.md)
> **Last reviewed**: 2026-07-04

**Rule.** Author REST APIs contract-first with a hand-authored OpenAPI spec as the single source of truth, generate Echo v4 Go delivery code from it with `oapi-codegen`, and never hand-write delivery shapes that diverge from the contract.

**Rationale.** [ADR-0018](../../adrs/0018-contract-first-rest-api.md) chose REST with a contract-first OpenAPI workflow so handlers, request/response DTOs, documentation, and future clients derive from one durable contract instead of drifting independently. Keeping generated HTTP shapes inside `delivery/http` preserves the existing Clean Architecture boundary: `app` and `domain` remain transport-neutral, and hand-written delivery code translates between generated DTOs and application inputs/outputs at the edge. This mirrors the [OIDC provider integration](../auth/oidc-provider-integration.md) precedent: external or generated details are allowed behind an adapter boundary, but they do not become inner-layer dependencies. REST keeps the public transport friendly to browsers, proxies, API gateways, generic HTTP tooling, and OpenAPI documentation without introducing a protobuf/gRPC runtime at the service boundary.

**Apply.**

- **Place the contract in the service API directory.** The hand-authored OpenAPI YAML lives under `services/<service>/api/*.yaml`; it is the source of truth for REST paths, operations, request bodies, response bodies, parameters, and schemas.
- **Generate Go only at the HTTP delivery boundary.** `oapi-codegen` output for a service belongs under that service's `internal/delivery/http/...` boundary, using the Echo v4 server generator. For physically split services such as portal, keep generated delivery packages aligned with the service's command/query HTTP layout (`internal/delivery/http/{command,query}/`) when that layout is used.
- **Mount generated routes on the shared Echo server.** Hand-written delivery wiring registers the generated Echo routes against the configured server exposed by `Echox.Echo()`; it does not bypass the shared `echox` server or reimplement competing route registration.
- **Translate at the boundary.** Generated request/response types are delivery DTOs. `internal/delivery/http/...` may depend on generated OpenAPI packages and on `app/{command,query}` ports, but `internal/app/...` and `internal/domain/...` must not import generated HTTP packages or expose generated types in use-case signatures.
- **Keep hand-written delivery code adapter-shaped.** Hand-written handlers, mappers, and middleware may satisfy generated interfaces, call application use cases, and translate app errors/results to generated responses; they must not define alternate request/response structs, route paths, or status semantics that compete with the OpenAPI contract.
- **Regenerate deterministically.** Generated files are derived artifacts: regenerate them from the checked-in YAML and checked-in generator configuration, keep generation stable across machines, and treat generated files as do-not-hand-edit output.
- **Carry ADR-0018 open questions forward.** This rule does not decide whether a service uses one `openapi.yaml` or split `command.yaml` + `query.yaml`, does not define the standard error-envelope shape, and does not define the API versioning strategy; those remain pending follow-ups in [ADR-0018](../../adrs/0018-contract-first-rest-api.md).

**Examples.**

✓ Good:
```text
services/portal/api/<contract>.yaml
  # Hand-authored OpenAPI source of truth.

services/portal/internal/delivery/http/<side>/openapi.gen.go
  # Generated by oapi-codegen from services/portal/api/<contract>.yaml.
  # Do not hand-edit.

services/portal/internal/delivery/http/<side>/handler.go
  # Hand-written adapter that satisfies the generated Echo server interface,
  # translates generated DTOs to app inputs, calls app/<side>, and maps results
  # back to generated responses.
```

```go
// delivery/http/<side>/routes.go — illustrative shape; generated names come
// from oapi-codegen.
func RegisterRoutes(server *echox.Echox, handler *Handler) {
	api.RegisterHandlers(server.Echo(), handler)
}
```

✗ Bad:
```go
// internal/app/query/list_staff.go — WRONG.
// The app layer now depends on generated HTTP DTOs from delivery.
import api "bootstrap/services/portal/internal/delivery/http/query"

type ListStaffUseCase interface {
	ListStaff(ctx context.Context, req api.ListStaffRequestObject) (api.ListStaffResponseObject, error)
}
```

```go
// internal/delivery/http/query/openapi.gen.go — WRONG.
// Manual edits to generated route/DTO code create a second source of truth.
func manuallyChangedStatusCode() int { return 299 }
```

**Enforcement.** Target enforcement is a named ADR-0018 follow-up: add deterministic devenv/CI codegen wiring that regenerates `oapi-codegen` output for every `services/*/api/*.yaml` contract and fails the build when generated files drift from the committed result. That automated gate is **not yet in place**. Until it exists, enforcement is PR review: reviewers must check that OpenAPI YAML is the source of truth, generated files are marked do-not-hand-edit and not manually modified, generated packages stay inside `internal/delivery/http/...`, `app`/`domain` do not import generated HTTP packages, and route registration mounts through `Echox.Echo()` rather than a parallel server path.
