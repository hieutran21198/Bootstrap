# Portal findings

The written record of investigations scoped to portal — debugging post-mortems, performance analyses, library evaluations, reproducibility checks. Workspace-wide or tooling investigations (e.g. the Claude/devenv agent config finding) live in [`docs/findings/`](../../../../docs/findings/).

> **Format, naming (`YYYY-MM-DD-<title>.md`), heavy-evidence layout (`<filename>.assets/`), and lifecycle are authoritative in the workspace track** — see [`docs/findings/README.md`](../../../../docs/findings/README.md). A `Resolved` finding is never re-edited; supersede it with a newer dated finding.

Start a new finding by copying [TEMPLATE.md](TEMPLATE.md) to `YYYY-MM-DD-<title>.md`.
